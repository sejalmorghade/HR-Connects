## Models Structure

### 1. **Profile** Model

- Links Django User to HR Connect roles
- Fields: role (student/hr), company, phone_number

### 2. **Student** Model

- Extends student information
- Fields: degree, branch, college, passing_year, skills, resume, contact, profile info
- Methods: get_resume_text() for PDF/TXT parsing

### 3. **Job** Model

- Job listings posted by HR
- Fields: title, company, skills_required, education_required, keywords

### 4. **Application** Model

- Student job applications
- Fields: student, job, match_score, applied_at

## URL Routing

Main URLs (HR_connect/urls.py):

- `/admin/` - Django admin interface
- All app URLs are included from app/urls.py

App URLs (app/urls.py):

- `/` - Home page
- `/register/` - User registration
- `/login/` - User login
- `/logout/` - User logout
- `/student/dashboard/` - Student dashboard
- `/student/update/` - Update student profile
- `/hr/dashboard/` - HR dashboard
- `/hr/post_job/` - Post job
- `/apply/<job_id>/` - Apply for job

## Configuration Details

### Settings Corrections Applied:

1. **TEMPLATES DIRS**: Now configured to include `BASE_DIR / 'templates'`
   - Enables shared templates at project root
   - APP_DIRS also enabled for app-level templates

2. **Static Files**:
   - STATIC_URL = 'static/'
   - STATIC_ROOT = BASE_DIR / 'staticfiles'
   - Custom static files go in: app/static/

3. **Media Files**:
   - MEDIA_URL = '/media/'
   - MEDIA_ROOT = BASE_DIR / 'media'
   - Resumes stored in: media/resumes/

4. **Database**:
   - Using SQLite3 (development)
   - Located at: BASE_DIR / 'db.sqlite3'

### Admin Interface (CORRECTED):

All models now registered in admin.py with proper configuration:

- **Profile**: Filtered by role, searchable by username/company
- **Student**: Filtered by college/branch/year, searchable by username/skills
- **Job**: Filtered by date/company, searchable by title/skills
- **Application**: Filtered by date/score, searchable by student/job, read-only protection

## Deployment Checklist

Before deploying:

1. Set DEBUG=False in settings.py
2. Use environment variables for SECRET_KEY and ALLOWED_HOSTS
3. Run `python manage.py collectstatic`
4. Use gunicorn (configured in requirements.txt)
5. Set up proper database (PostgreSQL recommended)
6. Configure ALLOWED_HOSTS properly
